# Polkadot Contraction DAPPS2024

[![DOI](https://img.shields.io/badge/DOI-10.1016/j.comnet.2026.112166-blue)](https://doi.org/10.1016/j.comnet.2026.112166) 

This repository holds the material related to the publication  
**"<a href="https://doi.org/10.1016/j.comnet.2026.112166">Toward estimating speculation in a cryptocurrency transaction network: The Polkadot case study</a>"**  
by *[Maurantonio Caprolu](https://orcid.org/0000-0001-8237-0539), [Roberto Di Pietro](https://orcid.org/0000-0003-1909-0336), [Flavio Lombardi](https://orcid.org/0000-0003-0723-7847), and [Elia Onofri](https://eliaonofri.it)*.   
The work is published in *Computer Networks* (**[ComNet](https://www.sciencedirect.com/journal/computer-networks)**), Volume 280, May 2026, 112166.

---

## Description

This repository contains the Python implementation carrying out the data analysis over the Polkadot network, as described in the above paper.  
The core script, `PolkadotStatistics.ipynb`, loads the contracted full graph from `data/full_graph/full_contracted_graph.gpickle` and the monthly contracted graphs from `data/monthly_graph/M<t>_contracted_graph.gpickle` to perform the analysis supporting all the results presented in the work, including plots and tables;
the mapping between Polkadot nodes' hashes and the cluster id within the present graphs are provided as `<graph_name>_H2V1.txt` files.
The additional market capitalisation data are stored in `data/capitalization`.


We refer to [this repository](https://github.com/m-caprolu/Polkadot-graph-analysis) for the technicalities regarding the extraction of the Polkadot full graph and
we refer to [this repository](https://github.com/eOnofri04/GraphColorContraction) for the details regarding the contraction of the graph.

_Note._ If you are interested in the results presented in this work, check also out our previous contribution [here](https://github.com/eOnofri04/Polkadot-DAPPS2024).  
[![DOI](https://img.shields.io/badge/DOI-10.1109/DAPPS61106.2024.00016-blue)](https://doi.org/10.1109/DAPPS61106.2024.00016)

## Python Dependencies

The main dependencies to run the code in this repository are:
 - `jupyter`            a server-based interface for python to run the code within Notebooks. 
 - `matplotlib`         a comprehensive library for creating static, animated, and interactive visualisations in Python
 - `numpy`              a fundamental package for scientific computing with support for multi-dimensional arrays and matrices
 - `pandas`             a powerful data analysis and manipulation library offering flexible data structures
 - `networkx`           a library for handling graph data structures
 - `mpl_chord_diagram`  a library to plot chord diagrams

_Note._ If `latex` is not installed on the computer, please deactivate its usage in the preamble of the main notebook by setting `useLatex = False`.

A convenient way to install Python libraries locally is to use Python's `virtualenv` package which also allows using `pip` package manager also on HomeBrew python installations.
A python virtual environment can be created via the following command
```sh
python -m venv <env_name>
```
where `env_name` is the name assigned to the virtual environment and to its corresponding folder.
Once created, virtual environments should be activated in order to be used and deactivated once used.
This can be done by sourcing `activate` file:
```sh
source env_name/bin/activate
# ...
deactivate
```
You can verify the correct activation as the prompt line should start with `(env_name)`
While the environment is active, `python3` and `pip` will refer to the corresponding environment executables, _i.e._ if `which`ed, you should obtain
```sh
which python3
/.../.../env_name/bin/python3
which pip
/.../.../env_name/bin/pip
```
To install libraries we can then use `pip` as
```sh
pip install --upgrade pip
pip install jupyter matplotlib numpy pandas networkx mpl_chord_diagram
```
To launch `jupyter` and visualise/interact with the notebook present in this repo, use
```sh
jupyter notebook
```

A complete list of the deps used during development can also be found in `pythonDeps.txt`
